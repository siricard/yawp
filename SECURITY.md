# Security Policy

## Reporting a Vulnerability

Please report security vulnerabilities privately to
<set-on-first-commit@example.org>. **Do not open public issues for security
reports.**

When reporting, please include:

- A description of the vulnerability and its impact.
- Steps to reproduce, or a proof-of-concept if available.
- Affected version(s) or commit SHA(s).
- Any mitigations or workarounds you are aware of.

We will acknowledge receipt of your report and work with you on a fix. The
project does not yet have a published response-time SLA; we will publish one
as the project matures.

Please do not disclose the vulnerability publicly until a fix has been
released and we have had a reasonable opportunity to coordinate disclosure
with you.
