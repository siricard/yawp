Yawp is a **decentralized, end-to-end encrypted communication platform** you can
run yourself. Identity is owned by users (cryptographic key pairs, not server
accounts), servers federate with each other, and any server can be self-hosted
on a VPS.

These docs serve three readers:

- **Self-hosters / operators** — you want to deploy and run a Yawp server.
  → [Self-hosting](/self-hosting/)
- **Curious & security-minded users** — you want to understand how identity,
  federation, and encryption actually work before you trust it.
  → [How Yawp works](/how-it-works/)
- **Contributors** — you want to build on the codebase.
  → [Contributing](/contributing/)

:::note
Vocabulary in these docs follows the project's canonical
[glossary](/reference/glossary/). The architecture rationale lives in the
[ADRs](/reference/adr/), which the concept pages link down to.
:::
